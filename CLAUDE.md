# CLAUDE.md — dealligence

Авторитетная память проекта. Читать первым каждую сессию.

## Что это
Движок семантического анализа B2B-переписки. Корпус → атомы → таймлайн →
State Vector сделки → политики/нарративы/ассистент. Наследник Smart Sales
по принципу, не по коду. Первый кейс — WhiteBIT. Переиспользуется для sales ops.

## Назначение и цели
- ЦЕЛЬ 1 — систематизация: корпус разобран на атомы, разложен в таймлайн,
  declared/observed расхождения детектируемы.
- ЦЕЛЬ 2 — решения: State Vector сделки → рекомендуемые действия, нарративы,
  ассистент переговоров.
- Переиспользуемость: тот же движок для sales ops (Aluprof и далее).

## Принципы (наследуются от Smart Sales)
- LLM только интерпретирует, факты не выдумывает.
- Каждый атом привязан к дословной цитате (raw_text). Нет цитаты — нет атома.
- Граница детерминизма: атомы и таймлайн фиксированы, выше — LLM-reasoning.
- Audit trail: каждое значение оси состояния несёт причину из атома.
- N=1: WB — один контрагент. Это онтология/каркас для решений, НЕ обучаемая
  политика. POMDP/RL из sales_ai не переносится. Переносится язык описания.

- Прямые вызовы api.anthropic.com / api.openai.com — ЗАПРЕЩЕНЫ. Только orchestrator.
- frv GitLab — READ-ONLY. Писать только в github `avrich5/dealligence`.
- Клиентские данные (корпус) в git НЕ коммитить. corpus_raw/corpus_txt/out в .gitignore.
