# dealligence

Движок семантического анализа B2B-переписки. Вход — разнородный корпус
(транскрипты, чаты, договоры, PRD). Выход — таймлайн атомизированных смыслов,
State Vector сделки, политики/нарративы/ассистент переговоров.
Наследник Smart Sales по принципу (facts→rules→state→LLM), не по коду.

Первый прогон: WhiteBIT. Вторая площадка на том же движке: sales ops.

## Порядок чтения
1. `CLAUDE.md` — память проекта, machine map, запреты, gates, цикл.
2. `concept/00_CONCEPT.md` — что строим, атом, онтология, детекторы.
3. `DEV_PLAN.md` — линии работ и цели.
4. `PROGRESS.md` — единый источник «где мы».
5. `tasks/` — единая точка чтения тасков для Claude Code.

## Структура
- `engine/` — код движка (ingest, atomizer, llm_client, detectors).
- `corpus_raw/`, `corpus_txt/` — корпус (git-ignored, клиентские данные).
- `out/` — атомы и производные (git-ignored).
- `config/` — онтология (канон типов/осей), пороги.
- `analysis/` — сходимость онтологии, таймлайн, кластеризация.
- `reports/` — session-отчёты, результаты детекторов.
- `concept/`, `inventory/` — аналитическая база.
- `tasks/` — таски для Claude Code.
- `_archive/` — устаревшее, не трогать.
