# DEV_PLAN — линии работ dealligence

Карта направлений. Авторитетный журнал — CLAUDE.md; концепт — concept/00_CONCEPT.md.

## Концепт (сжато)
Корпус B2B-переписки → атомы (дословно) → таймлайн → State Vector сделки →
политики/нарративы/ассистент. N=1 (WB), каркас решений, не обучаемая политика.

## Три линии работ

### Линия 1 — Систематизация (ЦЕЛЬ 1)
Корпус разобран, онтология сошлась, declared/observed детектируется.
- ingest всех форматов. Статус: ✅ (34 файла).
- атомизация всего корпуса. Статус: ✅ (2124 атома).
- сходимость онтологии → канон типов/осей. Статус: ⏳ TASK 02.
- таймлайн атомов. Статус: ⏳ TASK 03.
- детектор declared/observed. Статус: ⏳ TASK 04.

### Линия 2 — State Vector сделки (ЦЕЛЬ 1→2 мост)
Поверх канона и таймлайна: оси состояния со значениями из атомов, в динамике.
- causal_rules: ось ← атомы (по образцу Smart Sales, оси WB-специфичные). ⏳ TASK 05.
- State Vector на сегодня + траектория по таймлайну. ⏳ TASK 06.

### Линия 3 — Выходы (ЦЕЛЬ 2)
- рекомендуемые действия (политики над State Vector). ⏳
- генератор нарративов и контента. ⏳
- ассистент переговоров. ⏳

## Две цели
ЦЕЛЬ 1 = систематизация (линия 1 + State Vector). ЦЕЛЬ 2 = выходы (линия 3).

## Ключевое допущение (не переоткрывать без причины)
declared/observed расхождение — межфайловый детектор по таймлайну, не свойство
атома. Атомизатор фиксирует declared как видно в тексте. observed_absent
вычисляется сопоставлением declared-атома с отсутствием парного факта позже.

## Переиспользование
Движок доменно-нейтрален. Онтология эмерджентна → под sales ops перевыводится
из его корпуса, код не меняется. Это проверка переносимости (Aluprof — вторая площадка).
